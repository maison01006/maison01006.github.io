##### 목록

- source : 입력 형태 결정
- filter : 이벤트 처리 파이프라인 결정
- match : 출력 목적지 결정
- system : 시스템 광역 설정
- label : 내부 라우팅을 위해 Output, Filter를 그룹화
- @include : 다른 설정 파일 삽입

### source

- 데이터를 어떤 형식 으로 받을지 적는 명령어

- ```
  # Receive events from 24224/tcp
  # This is used by log forwarding and the fluent-cat command
  <source>
    @type forward
    port 24224
  </source>

  # http://<ip>:9880/myapp.access?json={"event":"data"}
  <source>
    @type http
    port 9880
  </source>

  <source>
    @type Input 플러그인
    path 파일 위치
    pos_file 읽어드린 파일의 내용을 어디까지 읽었는지 확인하는 파일 위치
    tag 태그
    <parse> 읽은 파일의 내용을 어떻게 파싱할지 정하는 태그
      @type parse 플러그인
  	expression 정규표현식
      time_key 시간 키를 설정
      time_format 시간포맷을 알려줌
      types id: 작성 형태 설정
    </parse>
  </source>
  ```

### filter

- 태그를 감지해서 형태를 변경

- ```
  <filter myapp.access>
    @type record_transformer
    <record>
      host_param "#{Socket.gethostname}"
    </record>
  </filter>

  <filter myapp.access> source에서 선언한 태그를 바탕으로 인식
    @type filter 플러그인
    <record>
      host_param "#{Socket.gethostname}"
    </record>
  </filter>
  ```

### match

- 태그를 감지해서 어떠한 형식으로 내보낼지 선언

- ```
  <match myapp.access>
    @type file
    path /var/log/fluent/access
  </match>

  <match myapp.access>  source에서 선언한 태그를 바탕으로 인식
    @type output 플러그인
    path 내보낼 위치 (플러그인에 따라 형식이 바뀜)
    <server> 어떠한 서버들로 로그를 전송할지 지정
    </server>
    <secondary> 전송에 실패한 로그에 대한 기록
    </secondary>
  </match>
  ```

### label

- 그룹화

- ```
  <label @SYSTEM>
    <filter var.log.middleware.**>
      @type grep
      # ...
    </filter>
    <match **>
      @type s3
      # ...
    </match>
  </label>
  ```

### @include

- 다른 설정파일 불러오기

- ```
  # absolute path
  @include /path/to/config.conf

  # if using a relative path, the directive will use
  # the dirname of this config file to expand the path
  @include extra.conf

  # glob match pattern
  @include config.d/*.conf

  # http
  @include http://example.com/fluent.conf
  ```

## 플러그인

### Input 플러그인

- tail : 새로운 파일만 처음부터 읽고 나머지는 tail를 읽어들인다.

  - ```
    <source>
      @type tail
      path /var/log/nginx/access.log
      pos_file /var/log/fluent/nginx-access.log.pos
      tag nginx.access
      <parse>
        @type nginx
      </parse>
    </source>
    ```

- forward : TCP로 데이터 수신한다. 다른 Fluentd 노드로부터 데이터를 전달 받을때 사용한다.

  - ```
    <source>
      @type forward
      port 24224
      bind 0.0.0.0
    </source>
    ```

#### Parser 플러그인

- parser_regexp : 정규표현식으로 데이터를 파싱할 수 있다. 최소 1개 이상의 캡쳐 그룹과 time 캡쳐 그룹이 필요하다. time 캡쳐 그룹의 키 이름은 time_key 파라미터로 변경할 수 있다.

  - ```
    <parse>
      @type regexp
      expression /^(?<remote_addr>[^ ]+) "(?<http_x_forwarded_for>([^ ]+(, )?)+)" (?<http_x_client>[^ ]+) \[(?<timestamp>(0?[1-9]|[12][0-9]|3[01])/[a-zA-Z]+/\d\d\d\d:(00|0[0-9]|1[0-9]|2[0-3]):([0-9]|[0-5][0-9]):([0-9]|[0-5][0-9]) \+[0-9]+)\] "(?<request_method>\S+) (?<request_uri>[^"]+) (?<server_protocol>[^"]+)" (?<status_code>\d{3}) (?<body_byte_sent>\d+) "(?<http_referer>[^"]*)" "(?<http_user_agent>.+)" (?<request_time>[^ ]+)$/
      time_key timestamp
      time_format %d/%b/%Y:%H:%M:%S %z
      timezone +09:00
    </parse>
    ```

- parser_none : 데이터를 행마다 새로운 필드 1개로 다시 담을 때 사용한다. 데이터를 가공하지 않고 다음 플러그인이나 다른 Fluentd 노드로 전달할 때 사용된다.

  - ```
    <parse>
      @type none
      message_key log  # JSON 형식의 "log" Key로 데이터가 담긴다
    </parse>
    ```

### Filter 플러그인

- filter_grep : 명시된 필드값에 정규표현식과 매칭되는 값만 필터링 한다.

  - ```
    <filter dev.postgresql.*>
      @type grep
      <regexp>
        key log
        pattern ^statement: .+$
      </regexp>
    </filter>
    ```
